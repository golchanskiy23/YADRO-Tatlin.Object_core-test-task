# Name-Frequency-Counter

Консольная утилита на Go для подсчёта частоты встречаемости имён из текстового файла.
Каждая строка файла — одно имя. Утилита читает файл, подсчитывает вхождения каждого имени и выводит результат в порядке убывания частоты (при равной частоте — в лексикографическом порядке по возрастанию).

Большие файлы обрабатываются параллельно через воркер-пул размером `runtime.GOMAXPROCS(0)`.

## Запуск

| Флаг       | По умолчанию | Описание                                                              |
|------------|--------------|-----------------------------------------------------------------------|
| `top`      | —            | Вывести только N первых записей по приоритету; 0 — не выводить ничего |
| `output`   | stdout       | Путь к файлу для записи результата вместо stdout                      |

Через Makefile:
```bash
# подсчитать все имена
make run FILE=names.txt

# вывести только топ-10
make run FILE=names.txt TOP=10

# записать результат в файл
./name-frequency-counter count names.txt --output result.txt

# справка
./name-frequency-counter count --help
```

## Архитектура

```
CLI (cobra) → RunCountWithWriter → Parser.Split() → WorkerPool.Run() → последовательный Pop() → Printer
                                                            │
                                            N горутин читают чанки параллельно
                                                            │
                                                    SafeMap.Increment()
                                                            │
                                                MaxPriorityQueue (heap)
```

**Parser** (`internal/parser`) — открывает файл, определяет размер через `os.Stat`,
делит на N чанков с выравниванием границ по `\n`, чтобы ни одно имя не было разрезано, читает блоками по 4096 байт, избегая затрат на системные вызовы при чтении по 1-му байту.

**WorkerPool** (`internal/counter`) — запускает N горутин через `sync.WaitGroup`,
каждая читает свой чанк и передаёт строки в `SafeMap`.

**SafeMap** (`internal/counter`) — потокобезопасная карта `имя → *Item`.
При первом вхождении создаёт `Item` и добавляет в очередь через `Push`.
При повторном — атомарно инкрементирует счётчик и восстанавливает порядок кучи
через `IncrementAndFix` (инкремент и `heap.Fix` под одним мьютексом, чтобы исключить гонку).

**MaxPriorityQueue** (`internal/queue`) — max-heap над `[]*Item`.
Сортирует по убыванию `Count`, при равенстве — по возрастанию `Name`.
Не является потокобезопасной сама по себе — синхронизация обеспечивается на уровне `SafeMap`.

**Printer** (`internal/printer`) — форматирует `Item` в строку `имя:количество`.

## Замечания

- Регистр имён сохраняется без изменений: `Миша` и `миша` считаются разными именами.
- Строки, состоящие только из пробельных символов, игнорируются.
- Имена должны содержать только буквенные символы Unicode и пробелы; остальные строки пропускаются.
- `--top 0` означает не выводить ничего; отрицательное значение — ошибка.
